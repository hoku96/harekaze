# 実行手順書: マスターテンプレートのv0プロジェクト化(Claude Code側)

対象ファイル: `master_template_harekaze.html`(デザイン下敷き) / `build_master_v0_project.py`(実行スクリプト)

## 前提

- CoworkのサンドボックスはAPI通信がブロックされるため、この作業はClaude Code(通常のインターネット接続がある環境)で実行する。
- v0のAPIキーは会話に貼らず、実行環境の環境変数として直接設定すること(以下の手順参照)。

## 手順(ToDo)

- [ ] 1. `master_template_harekaze.html` と `build_master_v0_project.py` を同じディレクトリに置く
- [ ] 2. `pip install requests` を実行
- [ ] 3. APIキーを環境変数に設定する

  ```
  export V0_API_KEY="v0のAPIキー"
  ```

- [ ] 4. スクリプトを実行する

  ```
  python3 build_master_v0_project.py
  ```

- [ ] 5. Step 1(疎通確認)が失敗した場合

  - v0ダッシュボード(v0.dev/chat/settings/keys)でキーの有効性を確認
  - 契約プランがAPI利用に対応しているか確認(前回調査時点で公式ページ上の対応プランは未確認のまま。第三者情報ではPremium/Team以上との記載あり)
  - 認証ヘッダー形式(`Authorization: Bearer <key>`)がv0側の想定と一致しているか、v0公式ドキュメントの最新記載と照合

- [ ] 6. Step 2(生成)が成功したら、出力された`webUrl`を開いて人間の目でレビューする(この段階では見込み客にまだ見せない)
- [ ] 7. `v0_master_project.json` に保存された`chatId`/`projectId`を控えておく。これが以降「店舗ごとにカスタマイズする際の複製元」になる

## 次工程(このマスタープロジェクトができた後)

1. リード(店舗)ごとに、このプロジェクトを`v0.chats.init()`で複製し、店舗名・業態・強みを差し替えるメッセージを送る
2. 生成結果を人間がレビュー
3. 承認後にデプロイして営業提示用URLを発行
4. 成約後にバックエンド(予約データ保存)を拡張

(詳細は先に作成した `agent_spec_draft.md` のStep1以降を参照)

## 注意

- Step 2以降はv0の生成AIが実際に動くため課金対象。テスト目的で何度も叩き直さないよう、Step1で疎通確認が通ってから実行すること。
- APIキーはリポジトリやチャットに残さないこと。作業後は必要に応じてv0ダッシュボードでキーをローテーションする。
